# Camerobot

个人机器人摄影师 Pro：一个“类人型陪伴 + 专业摄影 + 无人机 + 轻摇臂 +
移动储物”的机器人系统。

> 产品心智：摄影界的智能航空母舰。

## 1. 产品定位

Camerobot 不是“带摄像头的移动机器人”，而是一个能理解视觉语言、执行摄影动作、
管理拍摄设备的具身摄影操作系统。

它需要做到：

1. 用户上传图片或视频作为参考。
2. 系统识别主体、位置、姿态、构图、景别、透视、镜头感、光线和运动方式。
3. 机器人把这些视觉目标转成身体动作：移动底盘、升降身体、调整相机头、
   控制摇臂、控制灯光、调用无人机。
4. 机器人在实时画面中持续修正，直到复刻或优化目标画面。

核心使用场景：

- 个人写真：用户一个人也能拍出接近摄影师掌镜的照片。
- Vlog / 旅拍：机器人跟拍、构图、补光、收纳设备。
- 活动记录：在人群中安全移动，完成多角度拍摄。
- 产品拍摄：重复稳定的机位、灯光、摇臂运动。
- 创作者工作流：把参考图变成可执行的 shot plan。

### 1.1 可替代的独立摄影师工作范围

Camerobot 的第一目标不是立刻替代所有摄影师，而是替代一部分高度重复、
可标准化、可以通过实时画面闭环验证的独立摄影师工作。

优先可替代：

- 基础人像拍摄：构图、对焦、曝光、补光、连拍和挑选候选图。
- 旅拍跟拍：保持主体在画面中的比例和位置，完成稳定移动拍摄。
- Vlog 摄影助理：跟随、自动构图、提示录制状态、管理设备。
- 小型产品拍摄：固定风格、固定机位、固定灯光、多角度复拍。
- 活动花絮：在人群中安全移动，捕捉指定主体和关键时刻。
- 基础运镜：推、拉、环绕、升降、低到高 reveal 等标准动作。
- 摄影助理工作：补光、举灯、倒计时、提示姿势、显示预览。

暂时不作为第一阶段完全替代目标：

- 强依赖摄影师个人审美和现场创意判断的大片创作。
- 复杂商业布光和大型片场调度。
- 高风险无人机环境。
- 需要深度人际沟通、情绪引导和导演能力的拍摄。

软件上要把“替代摄影师”的能力拆成可测量指标：构图误差、主体跟踪稳定性、
出片成功率、用户是否能看懂反馈、拍摄后是否能快速确认结果。

## 2. 硬件概念拆解

### 2.1 底盘与身体

- 轮式移动底盘，支持室内和轻户外移动。
- 身体可以储存摄影附件、电池、滤镜、线材、道具和小件物品。
- 身体高度可调，服务于不同机位高度：
  - 伸缩升降柱：最适合作为第一版，结构明确、风险低、可维护。
  - 剪叉升降：稳定但体积更厚重。
  - 躯干内置线性滑轨：适合做相机高度微调。
  - 磁悬浮：概念感强，但工程风险和成本过高，不建议作为 MVP。
- 相机头和底盘之间需要减震与稳定结构，避免轮子移动影响画面。

### 2.2 脸部相机系统

脸部区域用于适配当前主流相机和镜头：

- 手机相机。
- 微单 / 无反相机。
- 运动相机。
- 未来自研一体化相机模块。

能力要求：

- 云台支持 pan / tilt / roll。
- 可控快门、录像、曝光、对焦。
- 如果相机生态允许，进一步控制变焦、ND、光圈和跟焦。
- 支持实时取景流回传，用于闭环构图。

### 2.3 眼睛显示屏

眼睛不只是拟人表情，也应该是拍摄反馈屏和预览窗口。

显示内容：

- 拍摄状态：准备中、对焦中、倒计时、拍摄完成、录像中、暂停。
- 预览窗口：刚拍完的照片、当前实时取景的小窗、构图辅助线。
- 反馈提示：主体太靠边、距离太近、请抬头、请后退一步、光线不足。
- 设备状态：电量、存储空间、相机连接、无人机连接、灯光状态。
- 安全状态：正在移动、请保持距离、急停已触发。
- 情绪表达：在不需要预览时显示眼神、表情和陪伴反馈。

设计原则：

- 拍摄前：显示倒计时、构图引导和是否已锁定主体。
- 拍摄中：显示录制状态、红点、剩余时间和稳定性提示。
- 拍摄后：显示“拍摄完毕”和可快速确认的预览图。
- 预览不能遮挡核心传感器，也不能让用户误以为机器人已经停止移动。
- 隐私模式下，眼睛屏幕需要明确提示是否正在录制或拍照。

### 2.4 单手轻摇臂

一只手承担轻量摇臂 / 小型 crane 的角色，不优先追求人手灵巧度，而优先追求：

- 安全。
- 平滑。
- 可重复。
- 可控负载。

典型动作：

- 低到高 reveal。
- 推进 / 拉远。
- 环绕主体形成视差。
- 俯拍。
- 举灯、举反光板或小型附件。

### 2.5 头顶“耳朵无人机”

头顶耳朵可以作为无人机舱的产品语言。工程上建议分阶段：

1. 先做外部无人机 API 集成。
2. 再做自动起飞、返航、媒体同步。
3. 最后再做物理收纳、充电和弹出式结构。

无人机承担：

- 航拍建立镜头。
- 顶拍。
- 大范围跟随。
- 与地面机器人镜头衔接。

### 2.6 轻便灯光

灯光模块需要服务“复刻光线”和“改善光线”：

- 机身 / 头部 / 手臂上的小型补光灯。
- 可控亮度。
- 可控色温。
- 可选 RGB。
- 可调光线方向。
- 后续支持可拆卸小灯，机器人指导用户摆放。

## 3. 软件核心目标

软件要把“参考画面”转成“机器人动作”。

必须支持：

- 上传图片 / 视频。
- 图像处理与识别。
- 主体识别和分割。
- 人体姿态 / 物体姿态识别。
- 构图分析：
  - 主体在画面中的位置。
  - 景别。
  - 头顶空间。
  - 留白。
  - 地平线。
  - 前景 / 中景 / 背景关系。
- 机位估计：
  - 相机高度。
  - 相机距离。
  - 拍摄角度。
  - 近似焦段 / 视场角。
  - 透视关系。
- 光线分析：
  - 主光方向。
  - 补光方向。
  - 逆光 / 轮廓光。
  - 阴影方向。
  - 软硬程度。
  - 色温。
  - 明暗对比。
- 动作规划：
  - 底盘移动。
  - 身体升降。
  - 头部云台。
  - 摇臂轨迹。
  - 灯光控制。
  - 无人机任务。
- 实时闭环：
  - 对比实时画面和目标画面。
  - 自动微调机位、云台、灯光和主体位置提示。
- 眼睛显示屏反馈：
  - 显示拍摄状态、拍摄完成、倒计时和预览图。
  - 显示构图和姿势提示，让被拍摄者能直接理解机器人意图。
  - 在录制、拍照和隐私模式下提供明确状态提示。

## 5. 软件模块设计

### 5.1 Creator App

用户入口，用来表达拍摄意图。

功能：

- 上传参考照片 / 视频。
- 输入自然语言：
  - “完全复刻这张构图。”
  - “主体位置一样，但背景更干净。”
  - “同样的光线，拍成竖版。”
  - “用摇臂做一个从低到高的开场。”
  - “无人机先建立环境，再切回机器人跟拍。”
- 选择拍摄模式：
  - 人像。
  - Vlog。
  - 产品。
  - 活动。
  - 航拍。
  - 运镜。
- 预览 shot plan。
- 手动覆盖关键参数。
- 一键急停。
- 管理素材和历史镜头。

### 5.2 Reference Analysis

把参考图/视频变成结构化拍摄指令。

输入：

- 图片或视频。
- 可选文字描述。
- 可选 EXIF / 相机元数据。

输出：

- 主体 mask 和 bounding box。
- 人体或物体姿态。
- 构图参数：
  - 主体中心点。
  - 主体占画面比例。
  - 头顶空间。
  - 画面留白。
  - 地平线。
  - 前中后景层次。
- 机位参数：
  - 景别：特写、中景、全身、大全景。
  - 角度：平视、仰拍、俯拍、顶拍。
  - 焦段类别：广角、标准、长焦。
  - 近似相机高度。
  - 近似相机距离。
- 光线参数：
  - 主光方向。
  - 阴影方向。
  - 光线软硬。
  - 色温。
  - 对比度。
- 视频运动参数：
  - 推、拉、摇、移、环绕、升降、跟随、航拍 reveal。

### 5.3 Scene Reconstruction

理解机器人当前所处空间，避免只会“看图”而不会“落地执行”。

传感器：

- 实时相机画面。
- 深度相机或双目。
- IMU。
- 轮式里程计。
- 可选 LiDAR / ToF。
- 可选 UWB / 视觉定位标记。

职责：

- 实时定位主体。
- 建立局部 3D 空间。
- 识别障碍物和禁行区域。
- 判断机器人可到达机位。
- 判断背景是否能与参考图对齐。
- 为运动规划提供安全边界。

### 5.4 Shot Planner

把视觉目标转成身体动作。

输出：

- 底盘目标位姿：`x`, `y`, `yaw`。
- 身体升降高度。
- 头部云台角度：`pan`, `tilt`, `roll`。
- 手臂轨迹。
- 相机设置。
- 灯光设置。
- 无人机任务。

约束：

- 人体安全距离。
- 最大移动速度。
- 最大手臂速度。
- 最大负载。
- 画面抖动容忍度。
- 电量。
- 地面条件。
- 室内外、隐私和无人机法规。

### 5.5 Closed-loop Framing

机器人不能只执行一次计划，而要像摄影师一样持续修正。

闭环过程：

1. 读取实时画面。
2. 与目标构图比较。
3. 计算误差：
   - 主体中心偏差。
   - 主体大小偏差。
   - 头顶空间偏差。
   - 地平线偏差。
   - 背景相对位置偏差。
   - 曝光和光比偏差。
4. 按尺度分配动作：
   - 大幅位置偏差：底盘移动。
   - 高度偏差：升降结构。
   - 小幅构图偏差：云台。
   - 运镜需求：手臂 / 底盘轨迹。
   - 光线偏差：灯光亮度、色温、方向。
5. 误差进入容忍范围后拍摄。

### 5.6 Body Control Layer

所有身体部位都应暴露成清晰、可测试、可模拟的控制接口。

控制域：

- `base`：移动、停止、旋转、跟随、回充。
- `lift`：设置高度、校准、锁定。
- `torso`：稳定、旋转、平衡。
- `head`：pan、tilt、roll、主体跟踪。
- `camera`：对焦、快门、录像、变焦、曝光。
- `arm`：设置姿态、执行轨迹、摇臂动作、举灯。
- `lights`：亮度、色温、颜色、模式。
- `storage`：开锁、上锁、库存传感、电池仓状态。
- `drone`：起飞、跟随、返航、入舱、同步素材。
- `safety`：急停、防撞区域、速度限制、人员保护。
- `display`：眼睛表情、拍摄状态、预览窗口、构图提示、隐私提示。

### 5.7 Face Display UI

眼睛显示屏需要作为独立 UI 模块接入机器人运行时，而不是简单播放动画。

核心状态：

- `idle_companion`：待机陪伴表情。
- `framing`：正在构图，显示取景小窗和辅助线。
- `countdown`：倒计时拍摄。
- `capturing`：正在拍照或录像。
- `capture_done`：拍摄完毕。
- `preview`：展示刚拍完的照片或视频关键帧。
- `guidance`：提示用户移动、转身、看镜头、调整姿势。
- `privacy`：明确显示是否正在录制、是否关闭上传、是否本地处理。
- `safety`：移动、防撞、急停等高优先级状态。

显示策略：

- 高优先级安全提示覆盖所有表情和预览。
- 拍摄完成后自动进入短时预览，用户可以通过 App、语音或机身按钮确认。
- 当机器人移动时，眼睛显示屏要同时表达“我正在移动”，避免用户靠近。
- 预览图可以是相机原图的低分辨率代理，正式素材仍保存在相机或媒体库。
- 如果脸部同时承担相机安装位，显示屏布局需要避开镜头、传感器和散热区域。

## 7. MVP 路线

### MVP 0：纯软件仿真

目标：证明“参考图 -> 分析 -> shot plan”的核心链路。

- 上传参考图。
- 分析主体、构图、景别、光线。
- 生成结构化 shot plan。
- 在 Web 控制台可视化机器人机位、相机高度、灯光方案。
- 用假硬件接口模拟身体控制。
- 模拟眼睛显示屏状态：构图中、倒计时、拍摄完毕、预览。

### MVP 1：移动相机机器人

目标：实现基础复刻构图。

- 轮式底盘。
- 伸缩升降结构。
- 手机或微单云台。
- 实时主体跟踪。
- 闭环构图。
- 基础补光。
- 眼睛显示屏显示拍摄状态和拍后预览。
- 急停和防撞。

### MVP 2：电影感运镜

目标：加入可控运动表达。

- 轻量摇臂手臂。
- 推、拉、环绕、升降预设。
- 平滑轨迹规划。
- 视频参考分析。
- 运镜复刻。

### MVP 3：无人机和多机位

目标：形成“地面机器人 + 空中无人机”的摄影系统。

- 外部无人机 API 集成。
- 无人机 shot planner。
- 起飞 / 返航 / 素材同步。
- 空中镜头与地面镜头衔接。

### MVP 4：Pro 级陪伴摄影师

目标：从工具升级为创作伙伴。

- 语音交互。
- 拍摄指导。
- 自动挑片。
- 自动套预设 / 初剪。
- 本地隐私模式。
- 云端素材同步。
- 工作室模式、旅拍模式、活动模式。

## 8. 关键技术风险

- 单张参考图反推真实 3D 机位并不总是唯一解。
- 光线复刻需要区分“画面效果”和“真实光源”，难度高。
- 机器人移动、升降、云台、手臂同时工作时容易带来画面抖动。
- 真人附近运行时，安全策略必须优先于镜头效果。
- 微单和镜头负载会显著影响结构、续航和稳定。
- 相机品牌生态碎片化，控制接口不统一。
- 室内无人机风险较高，法律和安全约束明显。
- 产品复杂度很大，必须先验证核心摄影闭环，再扩展炫酷硬件。

## 9. 第一版建议

建议第一台原型机选择：

- 伸缩升降，不做磁悬浮。
- 低速轮式底盘。
- 手机或轻量微单云台作为“脸”。
- 眼睛显示屏用于状态反馈、倒计时和拍后预览。
- 一个可控补光灯。
- 手臂先做小负载摇臂，不做人手级操作。
- 无人机先做软件集成，不做自动弹出结构。
- Web App 做上传、分析、shot plan、手动覆盖、预览确认和急停。

这样可以最快验证最重要的产品命题：

> Camerobot 能不能看懂一张参考图，并自动移动、升降、构图、补光，拍出
> 接近参考图的画面，同时让用户通过眼睛显示屏确认拍摄状态和结果？

## 10. 北极星能力

最终 Camerobot 应该能理解这些指令：

- “按这张图的主体位置和构图拍我。”
- “保持同样构图，我往前走你跟拍。”
- “先无人机航拍 reveal，再切回机器人地面跟拍。”
- “灯光太硬，变柔一点。”
- “重复昨天在这里拍的机位和光线。”
- “帮我拍一组产品图，构图统一但角度不同。”
- “拍完给我在眼睛上看一眼预览。”
- “如果构图不对，直接在屏幕上告诉我怎么站。”

长期目标是一个具身摄影系统：感知、构图、运动、灯光、相机控制、无人机协同、
素材管理和创意指导都在同一个机器人摄影师中完成。
